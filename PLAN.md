# Veloce アプリケーション詳細仕様・機能定義書 (PLAN.md)

本書は、AI生成画像に特化した超軽量・高速画像管理マネージャー＆ビューアー「Veloce」の、実際のソースコード構造および実装に基づいた正確な仕様書である。

---

## 1. システムアーキテクチャ・動作環境

### 1.1 技術スタック & コア最適化
* **バックエンド:** Rust (Tauri v1)
  * **高効率メモリ管理:** `mimalloc` (`MiMalloc`) をグローバルアロケータに指定し、長時間の画像処理における断片化とメモリ肥大化を徹底的に抑制。
  * **マルチスレッド処理:** `rayon` による並列ソート、および `tokio::task::spawn_blocking` によるI/Oバウンドな非同期タスク分離。
* **データベース (キャッシュレイヤー):** SQLite (r2d2接続プール接続、最大16スレッド)
  * **超高速設定:** `journal_mode = WAL`, `synchronous = NORMAL`, `cache_size = -64000 (約64MB)`, `temp_store = MEMORY` を指定。
* **フロントエンド:** Vite + Vanilla JS + HTML5 / CSS3
  * **UIフレームワーク不使用:** ReactやVueなどの仮想DOMは一切排除。生のDOM操作と Event Delegation（イベント委譲）、および DOM Pool 構造により極限まで軽量化。
* **ファイルハッシュ:** 暗号学的ハッシュではなく、極めて高速な非暗号学的ハッシュ `xxHash (xxh3_64)` をベースに「`パス_更新日時(mtime)`」から16進数16文字の `hash_key` を生成してキャッシュを一元管理。

### 1.2 動作環境要件
* **OS:** Windows 8.1 環境（WebView2 v109 / Chromium 109 互換）への完全準拠。
* **UI制約:** 文字化けや環境依存を防ぐため、システム内での絵文字（⭐や✖など）の使用は厳禁とし、すべて高品質な **SVGアイコン（`ICON_SVGS`）で代替**。
* **JS/CSS制約:** 古いChromium互換のため、最新のECMAScript配列メソッドやCSSネストなどの利用制限を遵守。

## 2. コア機能詳細

### 2.1 独立マルチビューアーウィンドウ
* **ウィンドウ制御:** メイン画面から完全に独立した子ウィンドウを高速展開。多重起動時はショートカット（`A` キー）でモニターの有効ワークエリアに合わせて**横一列に瞬時に自動整列**。
* **Windows 8.1 前面化ハック:** 整列時、バックエンド側で別スレッドから `SetForegroundWindow` および `SetWindowPos (HWND_TOPMOST)` をミリ秒遅延させて呼び出し、古いOS特有のウィンドウフォーカス奪取問題を物理的に解決。
* **高速画像展開:** 拡大縮小、回転（左右90度）、水平・垂直反転、等倍（100%）表示、アンシャープマスク等による低解像度画像のシャープ化フィルターに対応。

### 2.2 ファイルシステム & ドライブ検出
* **ネイティブドライブ検出:** Windows APIの `GetLogicalDrives` を直接呼び出し、接続されている物理ドライブ（`A:\`〜`Z:\`）のビットマスクをミリ秒以下で検出して動的にツリーへ展開。
* **遅延読み込みツリー:** フォルダ構造は一度に読み込まず、ユーザーの展開操作（トグルクリック/キーナビ）に応じて非同期で下位フォルダを取得（遅延ロード）。先回りで「さらに1つ下の階層（孫）」にサブフォルダが存在するかを裏でチェックし、中身が空であれば展開用矢印アイコンを自動的に非表示にする最適化を実装。

### 2.3 タブシステム（マルチタスク）
* **複数タブ管理:** 複数タブの並列起動、個別閉じ（中クリック対応）、コンテキストメニューからの「タブ複製」「他のタブをすべて閉じる」「右側のタブをすべて閉じる」を完全にサポート。
* **状態の永続化:** タブごとの閲覧パス、スクロール位置、適用されているソート/検索クエリ設定を `tabsState` として LocalStorage に常時保存し、アプリ再起動時に寸分狂わずに前回の状態を復元。

### 2.4 ナビゲーション・履歴
* **進む・戻る履歴:** タブごとに移動履歴（`history` 配列と `historyIndex`）を保持。マウスの拡張ボタン（進む・戻るボタン）のフックにも対応。
* **コンテキスト履歴メニュー:** 戻る・進むボタンの「長押し（300ms）」または「右クリック」により、履歴内のフォルダ名とお気に入りアイコン・カラーが反映されたメニューをポップアップ表示し、一階層ずつ戻る手間を省いて目的の履歴へジャンプ可能。

## 3. メタデータパース & インスペクター

### 3.1 網羅的なAI生成メタデータ解析ロジック
画像バイナリを自前で解析し、主要なAI生成プログラミング形式を完全に網羅。
1. **PNG形式:** `tEXt` / `iTXt`（Zlib圧縮対応）チャンクから Description, Comment, parameters などを抽出。ComfyUIの `workflow` / `prompt` 構造のフォールバックも実装。
2. **WebP / JPEG形式:** バイナリをメモリマップ（`memmap2`）で展開し、TIFF IFD構造をパースして `ImageDescription` や `UserComment`（UNICODE/ASCIIヘッダースキップ処理付き）、`Software` を高速抽出。
3. **Stealth形式 (隠しメタデータ):** EXIF等に痕跡がない場合、画像のアルファチャンネル（最下位ビット）から Column-Major Order (x -> y) でビットを抽出し、MSB first でバイト配列に変換。`stealth_pngcomp` (Gzip圧縮) および `stealth_pnginfo` (非圧縮) をデコードしてプロンプトを復元。
4. **NovelAI V4 / V4.5 特殊対応:** JSON内のマルチキャラクタープロンプト（`v4_prompt/caption/char_captions`）およびネガティブ側を自動パースし、フロントエンド側で `characterPrompts` 配列（`prompt`, `uc`）に再構造化して保持。
5. **A1111 / WebUI形式フォールバック:** parameters 内の `Steps: ` 等の文字列パターンを検出し、自動的に `rawParameters` フィールドに格納してパース。

### 3.2 フロントエンド・インスペクター & DOM Pool
* **DOM Pool（要素の再利用）:** インスペクターに大量のプロンプトタグやパラメータを描画する際、メモリ解放（GC）による一瞬のガタつきを完全に排除するため、`inspectorSectionPool` および `inspectorTagPool` を実装。画面更新時はDOM要素を一度も破棄せず、`display = 'none'` による非表示化と中身の `replaceChildren()` による書き換えだけで要素を高速に再利用。
* **パラメータバッジ色分け:** パラメータのサブ属性を解析し、`Inpainting`（青）、`Vibe Transfer`（紫）、`Character Reference`（オレンジ）、`Image to Image`（緑）の専用バッジを割り当て、一目で生成アプローチを識別可能。
* **検索ワード連動ハイライト:** メイン画面の検索バーに入力された複数キーワード（カンマ区切り）と完全に連動し、インスペクター内の該当単語（タグ）の枠線と背景をゴールド（`#ffcc00`）に発光させ、テキスト自体にも `<mark>` タグによるハイライトを適用。

## 4. サムネイル & パフォーマンス最適化

### 4.1 ハイブリッド・サムネイル生成パイプライン
WebView2 と Rust バックエンドの得意分野を組み合わせた、独自の往復バッチシステム。
1. **ファストパス表示:** ディレクトリ展開時、Rust側がSQLiteを高速に1本釣り検索（IN句の上限999に合わせてバッチチャンク分割してクエリ）。すでにキャッシュが存在するファイル（`has_thumbnail_cache == true`）は、独自カスタムプロトコル `veloce://?path=...&mtime=...` を経由して、バックエンドのSQLiteからバイナリ（Blob）を直接WebView2に超高速でストリーミング配信。
2. **ブラウザ側生成フォールバック:** キャッシュが未生成の画像は、WebView2のハードウェアアクセラレーション（`createImageBitmap` 経由）を利用して、メインスレッドを一切ブロックせずに Canvas 上で最大384x384ピクセルの高画質Jpeg（クオリティ0.85、背景色 `#1e1e1e` 固定）にリサイズ・エンコード。
3. **Rustバックグラウンド保存:** 生成されたBase64データをRust側のコマンド `saveThumbnail` へ非同期で投げ、SQLiteへ永続キャッシュ。キャッシュ完了後はJS側の巨大なBase64メモリを即座に破棄し、軽量なプロトコルURL（`veloce://`）に差し替えることで、フロントエンドのメモリフットプリントを極限まで低く保つ。

### 4.2 動画対応・ストリーミングサーバー
* **ネイティブ動画サムネイル抽出:** MP4動画ファイルのサムネイルを要求された場合、Windows環境では組み込みの `ffmpeg.exe`（存在する場合、非ウィンドウモード `CREATE_NO_WINDOW` で実行）または Windows ネイティブの `IShellItemImageFactory`（Shell API）を呼び出し、動画の1秒目のフレームからダイレクトにビットマップ（BGRA→RGBA変換）を抽出し、高速にPNGデータとしてフロントエンドに返却。
* **ローカルHTTP動画サーバー:** 大容量の動画（MP4）をWebView2でスムーズに再生するため、Rust側でランダムな空きポートを使って `TcpListener` をホストする軽量なオンデマンドHTTPサーバーが稼働。
* **2MB制限チャンクストリーミング:** 動画再生時、ブラウザ側からの `Range: bytes=...` ヘッダーを解析。バックエンド側でファイルのシーク位置（`SeekFrom::Start`）を調整し、**1回のリクエストに対する返却サイズを最大2MBに厳格に制限**して部分配信（`206 Partial Content`）を行うことで、数GBを超える動画でもメモリを一切圧迫せずに即時シーク・再生可能。

### 4.3 仮想スクロール（バーチャルスクロール）
* **通信ボトルネックの排除:** フォルダ内に数万枚の画像があっても、Rust側で状態（`all_files` / `filtered_files`）をSource of Truthとして保持し、JS側には総件数（`total_count`）のみを通知。JS側の仮想スクロールの描画領域に必要な分（例：100件ずつ）だけ、`get_items(offset, limit)` コマンドでピンポイントにスライスして取得。

## 5. スマートフォルダ & 高度なフィルタ

### 5.1 評価（レーティング）システム
* SQLite上の独立した `ratings` テーブルで管理され、ファイルパスを主キーとして1〜5の数値を保存（0はレコード削除）。
* レーティングの変更は即座にフロントエンドのグリッド上のバッジ、およびリストビューの行にリアクティブに反映。

### 5.2 スマートフォルダの検索条件・演算子仕様
ユーザーがUI上で自由に組み合わせ可能な条件ルール（`SmartFolderRule`）は、バックエンド側で完全な動的SQLクエリに変換され、インデックス（`idx_cache_path`, `idx_cache_mtime` 等）を活用して数ミリ秒で全ファイルを対象にフィルタリングを実行。

| 条件タイプ (`type`) | 対応演算子 (`operator`) | 処理・計算ロジック |
| :--- | :--- | :--- |
| **prompt** | `contains` / `not_contains` | `searchable_prompt LIKE ?` による部分一致/不一致 |
| **negative_prompt** | `contains` / `not_contains` | `searchable_negative_prompt LIKE ?` による部分一致/不一致 |
| **source** | `contains` / `not_contains` | `searchable_source LIKE ?` による部分一致/不一致 |
| **rating** | `>=`, `<=`, `==`, `!=` | `coalesce(r.rating, 0)` を用いた数値評価 |
| **width** / **height** | `<=`, `==`, `>=` | `c.width` / `c.height` に対する数値評価 |
| **aspect_ratio** | `portrait` | 縦長画像判定公式: `(width * 100 < height * 95)` |
| | `landscape` | 横長画像判定公式: `(width * 100 > height * 105)` |
| | `square` | 正方形判定公式: `(width * 100 >= height * 95 AND width * 100 <= height * 105)` |
| **path** | `in_folder` | 指定フォルダの**直下のみ**を対象とする判定:<br>`c.path LIKE ? AND c.path NOT LIKE ?/% AND c.path NOT LIKE ?\%` |
| | `under_folder` | 指定フォルダの**配下すべて（サブフォルダ含む）**を対象:<br>`c.path LIKE ?%` |

* **特殊組み込みID:** 条件が空の特殊フォルダとして、`fav_5`（星5）、`fav_4_plus`（星4以上）、`history`（`last_accessed` 降順の直近100件）のファストパスを完備。
* **実ファイル非依存動作:** スマートフォルダの読み込み時は、ファイルシステム上の実ファイルの存在有無をチェックせず（`is_file`の同期的確認をスキップ）、SQLiteキャッシュ内の情報のみをベースに高速一覧化。ファイルが存在しない古いキャッシュであっても、保存されていた `size` や `mtime` を遅延読み込みでそのまま表示可能（テストにより挙動を保証）。

## 6. ファイル・システム操作 & 堅牢性

### 6.1 高度なドラッグ＆ドロップ（移動・コピー・衝突解決）
* アプリ内のツリーやグリッドから他のフォルダ、もしくはお気に入りバーへのD&Dを完全制御。
* **衝突検出（`check_conflicts`）:** ファイルドロップ時、バックエンド側で非同期にターゲットディレクトリ内の同名ファイルの有無を一括チェック。衝突を検知した場合、フロントエンド側にモダンな「衝突解決ダイアログ」を表示し、ユーザーは「上書き」「スキップ（重複を除外して継続）」「キャンセル」を選択可能。
* **インテリジェント動作判定:** ドロップ先のドライブ（ルート）と移動元のドライブが同じであればデフォルト動作を「移動（`move`）」、異なるドライブ間であれば「コピー（`copy`）」に自動で切り替え（Ctrlキーで強制コピー、Shiftキーで強制移動）。

### 6.2 堅牢なファイルウォッチャー（変更監視ポーリング）
* 外部のエクスプローラやブラウザからの画像保存、ゴミ箱からの復元を検知するため、バックエンド側で1500ms間隔の軽量なフォルダポーリングスレッドを常時稼働。
* **自動クリーンアップ仕様:** ウォッチャーがファイルシステム上での「ファイル削除」を検知した際、あるいはユーザーがアプリ内からファイルをゴミ箱（`trash` クレート経由）に移動した際、そのファイルに関連付けられていた SQLite 内のサムネイルバイナリおよびメタデータキャッシュ、さらに古いファイルベースのキャッシュ（`Thumbnails/*.jpg`, `Metadata/*.json`）を**ハッシュキーベースでストレージから即座に自動完全削除**。塵が一切残らない自動クリーンアップ機構により、ユーザーのディスク容量の無駄遣いを完全に防ぐ。

### 6.3 タイムトラベル（Undoスタック）
* アプリ内で行われた操作履歴を `undoStack`（配列）にスタック。ユーザーが `Ctrl + Z` を押した際、直前の操作に応じて逆方向のネイティブ処理を実行する。
  * `RENAME_FOLDER` / `RENAME_FILE` → 元の名前にリネーム
  * `MOVE_FILE` → 元のディレクトリに移動
  * `COPY_FILE` → コピーによって生成されたファイルをストレージおよびRust状態から物理削除
* 取り消しが成功すると、UIとキャッシュが自動的に再同期される。

### 6.4 主要なグローバルショートカット・インタラクション一覧
* `0` 〜 `5`: 選択中の画像にレーティング（星の数）を設定 / 解除（複数選択対応）。
* `F2`: 選択中のファイル/フォルダの名前変更プロンプトを表示。
* `Delete`: 選択中のファイル/フォルダを安全にゴミ箱へ移動。
* `Ctrl + C`: 選択中の画像をクリップボードに物理コピー。コピー成功時、リスト行またはサムネイルアイテムの可視領域に対して**「シャッターフラッシュ（不透明度0.5→0の白レイヤーフェクト）」を適用**し、視覚的な操作感を向上。
* `Ctrl + A`: 現在のフォルダ内のすべての画像を全選択。
* `Ctrl + F`: 検索キーワード入力欄に即座にフォーカスを移動。
* `Alt + ← / →`: フォルダ移動履歴の「戻る」 / 「進む」。
* `Ctrl + Tab` / `Ctrl + Shift + Tab`: 左右のタブへの高速切り替え。
* `Esc`: モーダル、コンテキストメニュー、カスタム選択ドロップダウン、ヘルプ画面の強制クローズ。
